# 測試地區：澳門與香港

> 本產品以 **澳門** 與 **香港** 作為主要測試與首發市場。  
> 技術與產品邏輯與原計劃一致，僅需將關懷資源與營運重心改為兩地在地化。

---

## 一、為何可行

| 面向 | 說明 |
|------|------|
| **技術** | GPS、H3 地理索引、地圖與聚合引擎均為全球通用，無地域限制 |
| **安全框架** | 四層審核、三級響應、關懷資源中心架構不變，僅需替換為當地心理援助熱線與機構 |
| **語言** | 繁體中文為主，香港可加粵語用語在地化；澳門可保留繁中+部分葡文說明（若需要） |
| **冷啟動** | 澳門面積小、人口集中，較易達到臨界密度；香港人口基數大，利於後續擴展 |
| **法規** | App Store / Google Play 上架流程相同；需符合香港《個人資料（私隱）條例》及澳門《個人資料保護法》 |

## 三、測試階段對應

| 階段 | 原計劃 | 澳門＋香港調整 |
|------|--------|----------------|
| Phase 0 | 50 人內測 | 50 人內測（澳門＋香港邀請制，可各半或依團隊資源分配） |
| Phase 1 | 台北 500 人 | **澳門＋香港** 共 500 位種子用戶；預埋內容覆蓋澳門半島/氹仔/路環、香港港島/九龍/新界東/西 |
| Phase 2 | 台北 DAU 5,000 | **澳門＋香港** 合計 DAU 5,000（可訂澳門 DAU 約 500–800、香港約 4,200+） |
| Phase 3 | 台北 DAU 10,000 | 澳門＋香港 DAU 10,000 |
| Phase 4 | 多城市（高雄/台中/新北） | 可改為：**香港各區深化** ＋ **澳門全境** ＋ **台灣或廣東省** 作為擴展 |
| Phase 5 | 國際（東京/香港等） | 香港已為首發市場，國際擴展改為東京/台灣/東南亞等 |

## 四、臨界密度與核心區域

- **澳門**：面積約 33 km²，人口約 68 萬；建議核心區為澳門半島、氹仔。臨界密度門檻仍為 **> 15 則/km²/日**，達標後該區可關閉環境情緒層。
- **香港**：面積約 1,100 km²，人口約 740 萬；建議核心區為港島（中環/銅鑼灣）、九龍（尖沙咀/旺角）、新界東（沙田/大埔）等。同樣 **> 15 則/km²/日** 解鎖即時模式。

---

## 五、合規提醒

- **香港**：《個人資料（私隱）條例》— 需提供私隱政策、收集目的、查閱/更正權、投訴管道；建議 App 內提供「刪除帳號及資料」功能。
- **澳門**：《個人資料保護法》— 類似要求，需明確告知處理目的、保留期及權利。
- 兩地均無強制「實名制」要求，匿名註冊在符合上述前提下可行。
